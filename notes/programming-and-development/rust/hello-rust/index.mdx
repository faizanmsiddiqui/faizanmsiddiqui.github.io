# Hello, Rust!

Rust is a systems programming language that runs blazingly fast, prevents
segfaults, and guarantees thread safety. It is designed to be a safe,
concurrent, and practical language (as discussed
[**previously**](/notes/programming-and-development/rust/what-is-rust/) as well)
that supports functional and imperative-procedural paradigms.

## Setting Up Rust Environment

### Installation

To get started with Rust, we need to install it on our system. The recommended
way to install Rust is through `rustup`, a command-line tool for managing Rust
versions and associated tools.

#### Linux, macOS, and Windows Subsystem for Linux (WSL)

In Linux, macOS, and Windows Subsystem for Linux (WSL), Rust can be installed by
running following command in the terminal.

```shell title="Terminal"
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

#### Other Platforms and Installation Methods

To install Rust on other platforms like Windows or Unix, or using different
methods, official installation guide
[**Other Rust Installation Methods**](https://forge.rust-lang.org/infra/other-installation-methods.html)
can be referred.

### Toolchain management with `rustup`

Rust installation comes with `rustup`, a toolchain installer for Rust. `rustup`
allows managing multiple Rust toolchains (stable, beta, nightly) and switching
between them easily. It also helps in keeping Rust and its tools up to date.

Rust can be updated to the latest version using the following command:

```shell title="Terminal"
rustup update
```

More information about `rustup` can be found in the
[**`rustup` official documentation**](https://rust-lang.github.io/rustup/).

### Configuring the `PATH` environment variable

In the Rust development environment, all the tools are install in the user's
home directory. `rustc` and `cargo` are under `~/.cargo`, while `rustup` is
installed in `~/.rustup`.

The `rustup`, `rustc`, `cargo`, and other commands will be added to Cargo's bin
directory, located at `~/.cargo/bin`.

- `rustup` is the Rust toolchain installer.
- `rustc` is the Rust compiler.
- `cargo` is Rust's package manager and build system.

After installation, these tools need to be accessible from the command line.
This is usually done automatically by the installer, but if not, then need to
add the following line to shell configuration file (e.g., `~/.bashrc`,
`~/.zshrc`):

```shell title="Terminal"
export PATH="$HOME/.cargo/bin:$PATH"
```

### Verifying Installation

To verify that Rust is installed correctly, the following commands can be run in
the terminal to check the versions of `rustc`, `cargo`, and `rustup`
respectively:

```shell title="Terminal"
rustup --version
```

```shell title="Terminal"
rustc --version
```

```shell title="Terminal"
cargo --version
```

If the installation was successful, these commands will display the installed
versions of Rustup, Rust compiler, and Cargo.

### Uninstalling Rust

Rust can be uninstalled using the following command:

```shell title="Terminal"
rustup self uninstall
```

## First Program

### Writing Code

To write a simple "Hello, World!" program in Rust, create a new file named
`main.rs` and add the following code:

```rust title="main.rs"
fn main() {
    println!("Hello, World!");
}
```

The file name can be anything, but it is common to use `main.rs` while the main
entry point of the program should always be `main` function.

### Compiling

Following Rust compiler `rustc` command can be used to compile the Rust program:

```shell title="Terminal"
rustc main.rs
```

This will generate an executable file named `main` (or `main.exe` on Windows) in
the same directory.

### Running

Finally, the compiled program can be run using the following command:

```shell title="Terminal"
./main
```

And it will print `Hello, World!` to the terminal.
